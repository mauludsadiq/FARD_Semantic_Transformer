# FARD Semantic Transformer

A 7-layer linguistic hierarchy trained and run in FARD — a deterministic, content-addressed scripting language. Every forward pass produces a cryptographic receipt. Every training step is a witnessed execution.

## Architecture

7-layer hierarchy: Phoneme → Syllable → Morpheme → Word → Phrase → Semantic → Discourse

Residual MLP stack: h_0..h_6 with per-layer classification heads.
Each h_{k+1} = h_k + MLP(h_k). Each head predicts from its own h_k.

## Models

### Linguistic Classifier (FARD-trained)
- D=32, vocab=[34,256,256,256,50,256,256]
- Trained on corpus_v8c_remapped.ndjson (freq-remapped class IDs)
- 3000 samples, 150 epochs, 3-phase curriculum, class-weighted loss
- Weights: train/remapped_weights.json

### Operation Sequencer (FARD-trained)
- Input: 25-dim feature vector → op_logits[8] + tgt_logits[8]
- 500 samples, 20 epochs, no PyTorch
- Weights: train/op_seq_fard_weights.json

## What Was Wrong (Retracted)

Previous "7/7 STRONG" raw-accuracy result was majority-class exploitation.
Root cause: alphabetical word class ID assignment caused L3 WORD to collapse
to 2 classes after capping (IDs 32-34454, cap=64 → everything → class 63).
Fix: frequency-remapped corpus (src/remap_corpus.py).

Previous D=16 FARD model trained on 200 samples from unremapped corpus.
Those results are not valid. Correct model: train/remapped_weights.json.

## What Remains

- L4/L5/L6 are LEARNING, not STRONG (BalAcc 0.15-0.26)
- Corpus has only 250 unique words — limited semantic diversity
- D=32 PyTorch model not yet ported to FARD training
- Causal intervention run in PyTorch, not FARD
